Hybrid RAG 详解

训练数据

项目详情
语料来源倪海夏《天纪》系列公开课字幕文本
语料规模约 87 万字，涵盖紫微斗数全部 14 主星、12 宫位、40+ 格局的详细讲解
切块策略按空行/章节边界语义切块，每块 800-2000 字，共约 500 个语义块
分词引擎jieba 中文分词 + 紫微术语保护（"杀破狼""三方四正"等复合词不被拆散）

BM25 通道 (权重 0.6)

关键词精确匹配。统计查询词在每个语料块中出现的频率，对专有名词（星曜名、宫位名、格局名）做硬匹配。保证"命宫七杀"不会漏掉任何一段倪师原话。

Embedding 通道 (权重 0.4)

项目详情
模型paraphrase-multilingual-MiniLM-L12-v2 (Sentence-Transformers)
向量维度384 维
模型大小约 470MB (首次下载，后续缓存)
编码方式每个语料块生成 chunk 摘要（关键词 + 前 200 字）→ 编码为 384 维向量 → 存入 .rag_index/embeddings.npy
查询流程用户命盘 → 提取所有星曜/宫位/四化 → 构建查询文本 → 编码查询向量 → 与 500 个语料向量做余弦相似度 → 召回语义相关块

混合打分

final_score = BM25_score × 0.6 + cosine_similarity( query_vec, chunk_vec ) × 0.4

Top 20 → 按 token 预算截取 (max 6000 tokens) → 注入 prompt

AI 解读设计

参数值原因
模型DeepSeek V4中文命理术语理解最优，成本可控
Temperature0.3平衡命理推演严谨性 + 倪师语言风格
Max Tokens4096覆盖四层解读（本命+身宫+大限+建议）
反幻觉System Prompt 顶层熔断缺大限流年数据时强制声明，严禁捏造
证据锚定盘上有才算数每条判断必须引用具体星曜+宫位出处

核心约束

  • 话不说死："倾向"而非"注定"，"容易"而非"必然"。紫微不是宿命论
  • 三层分开：本命盘（出厂设置）→ 大限（十年运势）→ 流年（当年变化）
  • 格局分寸：格局强弱取决于三方四正吉煞配合，不只报名字
  • 不恐吓：煞曜用"留意""多花时间"表达，不制造恐慌
  • 服务端校验：免费对话次数 SQLite 计数，不可客户端绕过
